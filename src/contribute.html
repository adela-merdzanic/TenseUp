Use & contribute

TenseUp is open source under the MIT license. This page explains what you're free to do with it and how to propose a change - even if you've never contributed to a project before.

Use it, fork it

  • Use the app as much as you like - no account, no strings attached.
  • Fork the repository and adapt it to your own needs: every question lives as plain JSON in src/data/topics/, so the same code can host a quiz on any subject - operating systems, programming, anything. The README's "Adding a new topic" section describes the file format.
  • There is no build step and there are no dependencies - clone it, serve the folder with any static server, done.

Found a mistake?

If you spot a wrong answer, a typo, or a bug in the app:

  • open an issue describing the question and what looks wrong, or
  • fix the topic file in src/data/topics/ yourself and open a pull request - question fixes are very welcome.

Contribute improvements

Pull requests are welcome - bug fixes, new topics, new features. Two things to keep in mind:

  • keep it vanilla HTML/CSS/JS with no build step and no dependencies;
  • formatting is handled by Prettier - the repo ships a config, so if your editor formats on save, your code is already consistent.

How to propose a change

  1. Fork the repository on GitHub. A fork is your own copy that you can push to - anyone can clone this repo, but only collaborators can push branches to it.
  2. Clone your fork and create a branch: git checkout -b fix-question-typo
  3. Make the change - for question fixes, edit the topic file in src/data/topics/.
  4. Run the app locally (npx serve src, python -m http.server inside src/, or VS Code Live Server) and check your change in the browser.
  5. Push the branch to your fork and open a pull request against main, describing what you changed and why.
Small fix, like a typo in one question? You can edit the file directly on github.com - click the pencil icon on the file and GitHub creates the fork, branch and pull request for you.
